# 2025년 8월 29일 교육 내용

# AWS

# 스토리지

## 🧱 블록 스토리지 (Block Storage)
블록 스토리지는 데이터를 **일정한 크기의 조각(블록)**으로 나누어 각각 고유한 주소를 부여해 저장하는 방식입니다.

(비유) 레고 블록 상자
각 데이터 블록은 레고 조각과 같습니다. 운영체제는 이 조각들을 원하는 곳에 마음대로 배치하고 조합하여 하드 드라이브와 같은 하나의 큰 볼륨을 만듭니다. 데이터의 위치를 직접 제어할 수 있어 매우 빠르고 유연합니다.

특징:

고성능: 데이터를 수정할 때 해당 블록만 찾아 바꾸면 되므로 속도가 매우 빠릅니다.

유연성: 운영체제가 직접 파일 시스템을 설치하고 관리할 수 있어 활용도가 높습니다.

주요 사용 사례: 데이터베이스, 가상 머신(VM)의 하드 드라이브 등 고성능과 낮은 지연 시간이 필수적인 작업에 사용됩니다. (예: AWS EBS)

## 📂 파일 스토리지 (File Storage)
파일 스토리지는 우리가 컴퓨터에서 흔히 보는 폴더와 파일의 계층 구조로 데이터를 저장하는 방식입니다.

(비유) 서류 캐비닛
데이터는 특정 서랍(폴더) 안의 서류(파일) 형태로 정리됩니다. 파일을 찾으려면 '총무부 서랍 → 2025년 폴더 → 8월 결산 파일'과 같이 정해진 경로를 따라가야 합니다.

특징:

직관적: 폴더와 파일 구조는 사람에게 매우 익숙하고 이해하기 쉽습니다.

공유 용이: 여러 사용자가 동시에 파일에 접근하고 공유하기 편리합니다. (NAS - Network Attached Storage)

주요 사용 사례: 여러 사용자가 공유하는 파일 서버, 로컬 컴퓨터의 하드 드라이브 등 일반적인 파일 저장 및 공유 환경에 적합합니다. (예: AWS EFS)

## 📦 객체 스토리지 (Object Storage)
객체 스토리지는 데이터를 '객체(Object)'라는 독립적인 단위로 저장하며, 각 객체는 데이터 본체, 고유 식별자(ID), 그리고 상세한 메타데이터(속성 정보)로 구성됩니다.

(비유) 주차장의 발레파킹
데이터(자동차)를 주차장에 맡기면, 어디에 주차했는지 신경 쓸 필요 없이 고유한 보관증(ID)만 받습니다. 나중에 이 보관증만 제시하면 직원이 알아서 차를 찾아다 줍니다. 자동차의 종류, 색상, 주인 정보 등은 메타데이터로 관리됩니다.

특징:

무한한 확장성: 계층 구조가 없어 수십억 개의 객체를 저장해도 성능 저하가 거의 없습니다.

풍부한 메타데이터: 데이터에 대한 상세 정보를 포함할 수 있어 데이터 분석 및 관리에 유리합니다.

HTTP 기반 접근: 인터넷 주소(URL)를 통해 데이터를 쉽게 접근하고 공유할 수 있습니다.

주요 사용 사례: 대용량 데이터 저장 및 백업, 클라우드 기반 애플리케이션의 데이터(사진, 동영상 등), 빅데이터 분석, 아카이빙 등에 최적화되어 있습니다. (예: AWS S3)

## 한눈에 보는 정리
| 스토리지 방식	| AWS 대표 서비스 | 주요 특징 |
|---|---|---|
| 객체 스토리지	| S3 | 무한한 확장성, 인터넷으로 데이터 접근(URL), 백업/빅데이터 | 
| 블록 스토리지	| EBS | EC2의 하드 드라이브, 고성능, 데이터베이스 등 빠른 작업용 |
| 파일 스토리지	| EFS | 여러 서버에서 파일 공유, 폴더/파일 계층 구조 |


Amazon S3 스토리지 클래스는 데이터의 접근 빈도와 보관 기간에 맞춰 비용과 성능의 균형을 맞출 수 있도록 제공되는 다양한 데이터 저장 계층(Tier)입니다.

모든 물건을 똑같은 방식으로 보관하지 않는 것처럼, 데이터도 사용 방식에 따라 다르게 보관하여 비용을 최적화하는 개념입니다. 집 안의 물건 보관에 비유하면 쉽게 이해할 수 있습니다.

## 🛋️ 자주 사용하는 데이터 (Hot Data)
매일 사용하는 물건처럼 빠르고 즉각적인 접근이 필요할 때 사용합니다.

S3 Standard
(비유) 거실에 둔 물건

가장 기본적이고 일반적인 스토리지 클래스입니다. 어떤 데이터든 즉시 꺼내 쓸 수 있도록 항상 대기하며, 가장 빠른 속도와 높은 내구성을 제공합니다. 웹사이트 이미지, 모바일 앱 데이터 등 자주 접근하는 원본 데이터 저장에 적합합니다. 저장 비용은 가장 비싸지만, 데이터를 꺼내는(검색) 비용은 저렴합니다.

S3 Intelligent-Tiering
(비유) 스스로 정리되는 스마트 보관함

데이터의 접근 패턴을 AWS가 자동으로 분석하여, 자주 사용하면 Standard 층으로, 뜸하게 사용하면 저렴한 Infrequent Access 층으로 자동 이동시켜주는 지능형 클래스입니다. 데이터 접근 빈도를 예측하기 어려울 때 사용하면 별다른 고민 없이 비용을 최적화할 수 있어 매우 유용합니다.

## 📦 가끔 사용하는 데이터 (Warm/Cool Data)
자주 쓰지는 않지만 필요할 때 바로 꺼내야 하는 데이터에 적합합니다.

S3 Standard-IA (Infrequent Access)
(비유) 집 창고에 넣어둔 물건

접근 빈도는 낮지만 필요할 때는 Standard처럼 빠른 속도로 접근해야 하는 데이터를 위한 클래스입니다. 저장 비용은 Standard보다 저렴하지만, 데이터를 꺼낼 때마다 데이터 검색 비용이 부과됩니다. 로그 파일, 백업 데이터 등에 적합합니다.

S3 One Zone-IA
(비유) 창고 중에서도 저렴한 한쪽 칸

Standard-IA와 거의 동일하지만, 데이터를 단 하나의 가용 영역(AZ)에만 저장하여 비용을 더 낮춘 클래스입니다. 따라서 해당 AZ에 재해가 발생하면 데이터가 유실될 수 있습니다. 중요도가 낮은 데이터의 백업이나, 쉽게 다시 생성할 수 있는 데이터 저장에 사용됩니다.

## 🧊 거의 사용하지 않는 데이터 - 아카이브 (Cold Data)
법규 준수나 기록 보존을 위해 장기간 보관만 하고 거의 접근하지 않는 데이터를 위한 클래스입니다.

S3 Glacier Family
(비유) 집에서 멀리 떨어진 저렴한 컨테이너 창고

저장 비용이 매우 저렴한 대신, 데이터를 꺼내는 데 시간과 비용이 더 많이 듭니다.

Glacier Instant Retrieval: 수 밀리초 내에 검색 가능. (IA와 Glacier 사이)

Glacier Flexible Retrieval: 몇 분에서 몇 시간 소요. 일반적인 아카이브용.

Glacier Deep Archive: 12시간 이상 소요. 가장 저렴하며, 거의 꺼낼 일이 없는 데이터 보관용.


AWS는 데이터의 종류, 규모, 네트워크 상태 등 다양한 마이그레이션 시나리오에 맞춰 여러 전문 도구를 제공합니다. 이 도구들은 크게 온라인(네트워크) 방식과 오프라인(물리적 장비) 방식으로 나뉩니다.

'어플라이언스'는 이 중 오프라인 방식에서 사용되는 물리적인 데이터 전송용 하드웨어 장비를 의미합니다.

## AWS 데이터 마이그레이션 도구
### 🌐 온라인 네트워크 전송 도구
네트워크를 통해 데이터를 직접 AWS로 전송하는 방식입니다.

AWS DataSync

온프레미스(사내) 파일 서버(NFS, SMB)나 다른 클라우드 스토리지의 데이터를 AWS S3, EFS, FSx 등으로 빠르고 안정적으로 복사하는 자동화 서비스입니다. 암호화, 데이터 검증, 스케줄링 기능을 제공하여 대규모 파일 전송에 최적화되어 있습니다.

AWS Transfer Family

SFTP, FTPS, FTP와 같은 표준 파일 전송 프로토콜을 사용하여 데이터를 S3나 EFS로 직접 업로드할 수 있게 해주는 완전 관리형 서비스입니다. 기존의 파일 전송 워크플로우를 변경하지 않고 클라우드로 확장할 때 유용합니다.

AWS DMS (Database Migration Service)

데이터베이스를 AWS로 마이그레이션하는 데 특화된 서비스입니다. Oracle, MySQL 등 기존 데이터베이스를 AWS의 RDS나 Aurora로 이전할 때 사용됩니다. 다운타임을 최소화하면서 실시간에 가까운 데이터 복제가 가능합니다.

## 🚚 어플라이언스란? (The AWS Snow Family)
어플라이언스는 네트워크 속도가 느리거나, 데이터 규모가 너무 커서(수십 테라바이트 이상) 네트워크로 보내는 것이 비효율적일 때 사용하는 물리적인 데이터 운송 장비입니다. AWS가 안전한 하드웨어 장치를 사용자에게 배송해주면, 사용자는 여기에 데이터를 복사해서 AWS로 다시 보내는 방식입니다.

마치 우편으로 작은 소포를 보내는 대신, 이사 트럭을 불러 한 번에 짐을 옮기는 것과 같습니다. 이 어플라이언스 제품군을 **'AWS Snow Family'**라고 부릅니다.

### 1. AWS Snowcone
(비유) 견고한 서류 가방

가장 작고 가벼운 휴대용 장치입니다. 약 8TB의 데이터를 저장할 수 있으며, 크기가 작아 드론, 차량 등 공간이 협소한 환경이나 소규모 데이터 이동에 적합합니다.

### 2. AWS Snowball Edge
(비유) 대형 이삿짐 박스 또는 여행용 캐리어

가장 널리 사용되는 어플라이언스입니다. 보통 80TB 내외의 데이터를 저장할 수 있는 '스토리지 최적화' 모델과, 컴퓨팅 파워가 더 강화된 '컴퓨팅 최적화' 모델이 있습니다. 데이터 센터 이전이나 대용량 데이터 수집에 주로 사용됩니다.

### 3. AWS Snowmobile
(비유) 40피트 컨테이너 트레일러

엑사바이트(Exabyte) 규모의 데이터를 옮기기 위한 궁극의 솔루션입니다. **100PB(페타바이트)**까지 저장할 수 있는 컨테이너 트럭이 직접 데이터 센터로 찾아옵니다. 초거대 데이터 센터를 통째로 클라우드로 옮기는 수준의 마이그레이션에 사용됩니다.
